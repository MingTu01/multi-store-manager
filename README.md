# 多店管理系统 (Multi-Store Manager)

> 适用于多门店经营的综合管理平台，支持桌面端和移动端PWA，iOS原生UI风格

## 版本：v0.4.5

## 版本规范

- **版本格式**：`v主版本.次版本.修订号`（如 v0.3.0）
- **当前阶段**：开发测试期，主版本号保持 `0.x.x`
- **上线前**：未经老大确认，**禁止**将主版本号升到 `1.0.0` 或更高
- **次版本号**：新增功能时递增（如 v0.3.0 → v0.4.0）
- **修订号**：Bug修复时递增（如 v0.3.0 → v0.3.1）

## 技术栈

- **后端**：Express 5 + TypeScript + SQLite（WAL模式）
- **前端**：Vite + React 19 + TypeScript + Tailwind CSS 4
- **OCR**：Tesseract.js（离线中文识别）
- **拖动排序**：@dnd-kit
- **端口**：3001（可通过 PORT 环境变量修改）

## 功能模块

### 管理端（ADMIN）
- **管理大屏** — 全店收支总计、环比同比、收入支出构成饼图、门店对比
- **门店管理** — 创建/编辑/删除门店、多张照片、股东信息（强制100%份额）、员工数量
- **操作日志** — 全局操作记录，支持按门店/时间/操作类型筛选+IP记录
- **系统设置** — 数据备份恢复、ZIP升级、消息推送配置、权限说明
- **资金余额**（仅"总"页面）— 初始资金 + 收入 - 支出

### 门店端（按角色权限）
- **总览** — 今日收支、待办提醒（未确认工资/未盘点）、最近操作记录、快捷操作
- **记账** — 收入支出记录、分类管理、关键词搜索、分类筛选、金额区间筛选
- **盘点** — 物品条目管理、dnd-kit拖动排序、领出功能、盘点模式、库存同步、差异标注
- **开闭店** — 拍照确认（多张）、交接内容、历史记录
- **报表** — 收入支出构成、同比环比、对比柱状图
- **员工管理** — 员工信息、头像拍照/上传、岗位、工资设定
- **工资** — 自动生成工资单、修改确认、工资条（含员工照片和岗位）
- **分红** — 按股东占比自动计算分红、可分红余额显示
- **日志** — 门店操作记录
- **账户** — 修改密码、个人信息、健康证上传+OCR识别、到期提醒

### 健康证管理
- **上传** — 拍照或选择文件，弹窗选择方式
- **OCR识别** — Tesseract.js离线中文识别，自动提取姓名和体检日期
- **误识别纠正** — 常见OCR误识别自动修正（月→户、日→扩等）
- **手动输入** — 识别不完整时可手动填写，有效期自动计算（体检日期+1年）
- **到期提醒** — 提前30天每周提醒，到期后每天提醒
- **图片展示** — 点击放大查看，object-contain完整显示

### 消息推送
- **支持通道**：PushPlus / Server酱 / 企业微信自建应用（需代理）
- **推送内容**：每日简报、每周简报、月度报告、待审核提醒、系统告警
- **模板风格**：纯文本 + Unicode符号排版，微信原生显示

### 安全特性
- JWT认证，密钥通过环境变量配置
- 门店级数据隔离（后端中间件校验）
- 角色权限前后端双重校验
- 登录速率限制（express-rate-limit，每分钟10次）
- 路径遍历防护、代码注入防护
- 图片上传自动压缩+仅限图片验证
- CORS可配置、JSON body大小可配置

### 数据可见性规则
- **管理员**：看到全部记录（含系统自动生成的工资/分红支出）
- **店长/员工**：只看到手动录入的记录（隐藏系统记录）
- **员工**：只看今日记账，隐藏利润卡片，只能新增不能修改删除
- **报表数据**：包含系统记录（保证金额正确），但明细不暴露

## 环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| JWT_SECRET | multi-store-secret-key-2024 | JWT密钥，**必须**修改 |
| TOKEN_EXPIRY | 24h | Token有效期 |
| CORS_ORIGIN | * | CORS允许的来源域名 |
| JSON_LIMIT | 30mb | JSON请求体大小限制 |
| PORT | 3001 | 服务端口 |
| NODE_ENV | (无) | 设为 production 时隐藏错误详情 |

## 部署

### 启动
```bash
cd apps/server && node --import tsx src/index.ts
```

### 构建前端
```bash
cd apps/web && rm -rf dist && npx vite build
```

### 打包升级ZIP
```bash
cd apps/server && node ../build-upgrade.js
```
生成 `upgrade-vX.X.X.zip`，在系统设置 → 系统升级页面上传即可。

### 升级流程
1. 上传ZIP包 → 自动验证包结构
2. 自动备份当前数据库
3. 解压覆盖 server-src 和 web-dist
4. 更新版本信息
5. 页面自动刷新到最新版本

### 数据备份
- 自动定时备份（可配置频率）
- 手动备份（ZIP下载）
- 升级前自动备份
- 一键恢复（含重启）
