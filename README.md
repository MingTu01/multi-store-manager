# 多店管理系统 (Multi-Store Manager)

> 适用于多门店经营的综合管理平台，支持桌面端和移动端PWA，iOS原生UI风格

## 版本：v0.5.0

## 版本规范

- **版本格式**：v主版本.次版本.修订号（如 v0.3.0）
- **当前阶段**：开发测试期，主版本号保持 `0.x.x`
- **上线前**：未经老大确认，**禁止**将主版本号升到 `1.0.0` 或更高
- **次版本号**：新增功能时递增（如 v0.3.0 → v0.4.0）
- **修订号**：Bug修复时递增（如 v0.3.0 → v0.3.1）

## 技术栈

- **后端**：Express 5 + TypeScript + SQLite（WAL模式）
- **前端**：Vite + React 19 + TypeScript + Tailwind CSS 4
- **OCR**：Tesseract.js（离线中文识别）
- **拖动排序**：@dnd-kit
- **端口**：3001（可通过 PORT 环境变量修改）

## 功能模块

### 管理端（ADMIN）
- **管理大屏** — 全店收支总览、环比同比、收入支出构成饼图、门店对比
- **门店管理** — 创建/编辑/删除门店、多张照片、股东信息（管理员可见）、员工数量
- **系统设置** — 数据备份恢复、ZIP升级、消息推送配置（全局汇总推送）、权限说明
- **账户管理** — 修改个人信息、密码、退出登录
- **资金余额**（仅"总"页面）— 初始资金 + 收入 - 支出 - 工资 - 分红

### 店铺端（ADMIN / STORE_ADMIN / MANAGER / STAFF / SHAREHOLDER）
- **门店总览** — 快捷操作（记账/盘点/开闭店/报表）、今日/本月/本年数据
- **记账** — 收入/支出记录、分类管理、日/周/月/年/总筛选、日期选择器
- **盘点** — 添加条目（含拍照）、开始盘点（逐条盘点）、消耗+实际=库存、状态标记（正常/差异/丢失/报废/待补货）
- **开闭店** — 霸屏模式、拍照强制、交接内容、历史记录
- **报表** — 收入支出构成饼图、环比同比对比图、柱状趋势图、资金余额
- **工资** — 生成工资条（自动带出底薪+奖金-扣款）、确认发放、个人工资单弹窗（键盘左右/滑动切换）
- **分红** — 按股份比例自动计算、确认分红、个人分红单弹窗
- **员工管理** — 添加/编辑/删除员工、头像上传、岗位/月薪/角色/状态管理
- **日志** — 操作日志记录（记账/盘点/开闭店/工资确认/分红确认等）
- **设置** — 门店信息编辑（名称/地址/照片/股东）、分类管理、消息推送（单店独立配置）

### 消息通知系统
- **全局推送**（管理页面配置）— 全店汇总数据推送
- **单店推送**（店铺页面配置）— 单店数据推送，每店独立配置
- **推送渠道**：PushPlus / Server酱 / 企业微信自建应用
- **推送内容**：每日简报、每周简报、每月简报、审核提醒、异常警告
- **站内通知** — 通知角标、已读/未读、48小时自动清除
- **健康证OCR** — Tesseract.js识别、有效期管理、到期提前通知

## UI 风格

- iOS 原生风格：毛玻璃、半透明、圆角、高饱和
- 桌面端：左侧导航栏，完整布局
- 移动端：底部导航栏（最多5个+更多），PWA 支持
- 金额超1万显示"X.XX万"，点击切换纯数字
- 弹窗居中显示

## 部署

### 环境要求
- Node.js 18+
- 服务器：推荐 CentOS + 1Panel

### 安装
```bash
cd apps/server
npm install
node --import tsx src/index.ts
```

### 构建前端
```bash
cd apps/web
rm -rf dist
npx vite build
```

### 系统升级
1. 在系统设置 → 系统升级页面上传 ZIP 升级包
2. 系统自动备份 → 解压 → 更新版本 → 覆盖文件 → 完成
3. 点击确认自动刷新到最新版本

### 数据备份
- 手动备份：系统设置 → 数据备份 → 创建备份
- 自动备份：支持每小时/每天/每周频率
- 恢复备份：上传备份文件或选择已有备份恢复
