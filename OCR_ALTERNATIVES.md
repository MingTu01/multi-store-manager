# OCR 方案对比分析

## 当前方案：Tesseract.js（chi_sim+eng）

| 维度 | 评分 | 说明 |
|------|------|------|
| 中文识别准确率 | ★★★☆☆ | 印刷体约 85-90%，手写体差 |
| 速度 | ★★☆☆☆ | 首次加载语言包约 3-5 秒，识别约 2-3 秒 |
| 离线可用 | ★★★★★ | 完全离线，无需网络 |
| 部署复杂度 | ★★★★★ | npm install 即可 |
| 成本 | ★★★★★ | 免费开源 |

## 替代方案对比

### 方案 A：保持 Tesseract.js + 优化正则（推荐）
- **改动量**：最小，只改正则
- **效果**：姓名准确率从 71% 提升到接近 100%
- **风险**：无
- **适合**：当前阶段，健康证格式相对固定

### 方案 B：百度 OCR API
- **中文准确率**：★★★★★（99%+）
- **速度**：★★★★★（<1秒）
- **成本**：免费额度 1000 次/月，超出 0.003 元/次
- **缺点**：需要网络，需要注册百度智能云，需要管理 API Key
- **适合**：对准确率要求极高的场景

### 方案 C：腾讯云 OCR
- **中文准确率**：★★★★★（99%+）
- **速度**：★★★★★（<1秒）
- **成本**：免费额度 1000 次/月
- **缺点**：同百度，需要网络和 API Key

### 方案 D：PaddleOCR（本地部署）
- **中文准确率**：★★★★☆（95%+）
- **速度**：★★★★☆（需要 Python 环境）
- **成本**：免费开源
- **缺点**：需要 Python + PaddlePaddle，部署复杂度高

## 结论

**当前不需要换 OCR 方案。** 原因：
1. 健康证是标准格式，文字清晰，Tesseract 足够
2. 正则修复后准确率预计 100%
3. 已有手动输入预案，OCR 失败时用户可手动填写
4. Tesseract 完全离线，不依赖外部服务
5. 健康证上传频率极低（每人每年一次），不需要高性能

**如果未来需要更高准确率**（比如识别手写体、模糊照片），建议接入百度 OCR API 作为 Tesseract 的降级方案：Tesseract 优先，识别不到时提示用户选择"云端识别"或"手动输入"。
