## 一、项目概况

多店铺管理系统（Multi-Store Manager）是一个面向多门店经营的综合管理平台，支持桌面端和移动端PWA。

**技术栈：** Express + SQLite + React 18 + Vite + TypeScript + Tailwind CSS

**当前状态：** 系统已基本可用，核心功能完整，安全加固已完成主要工作。

### 剩余安全风险

| 风险 | 严重程度 | 说明 |
|------|----------|------|
| 登录无速率限制 | 高 | 无暴力破解防护，建议添加 express-rate-limit |
| 默认弱密码 | 高 | admin/123456，建议强制首次改密 |
| Token过期无Refresh | 中 | 24h后需重新登录，可加Refresh Token |
| 部分路由缺角色校验 | 中 | dashboard/logs/handovers 等路由缺后端角色检查 |
| CORS默认* | 低 | 可通过环境变量收紧，但默认开放 |

## 五、缺失功能分析

### 高优先级（影响日常使用）

| 功能 | 说明 | 影响 |
|------|------|------|
| **数据导出** | 记账/工资/分红导出Excel或PDF | 老板需要打印报表给合伙人，员工需要工资条凭证 |
| **登录速率限制** | 防暴力破解 | 安全基础，实现简单 |
| **首次登录强制改密** | 默认密码123456不安全 | 安全基础 |
| **记账批量操作** | 批量删除/批量修改分类 | 数据量大时逐条操作效率低 |

### 中优先级（提升体验）

| 功能 | 说明 | 影响 |
|------|------|------|
| **深色模式** | 系统级暗色主题 | 夜间使用体验 |
| **数据统计图表** | 门店对比柱状图、趋势折线图 | 管理决策需要 |
| **多语言支持** | 当前仅中文 | 国际化需求 |
| **操作撤销** | 记账误删后可恢复 | 数据安全 |
| **高级搜索** | 全局搜索（跨模块） | 数据量大时查找困难 |

### 低优先级（锦上添花）

| 功能 | 说明 | 影响 |
|------|------|------|
| 单店数据导出 | 按门店导出Excel | 方便门店独立管理 |
| 移动端手势优化 | 左右滑动切换页面 | 移动体验 |
| 离线模式增强 | PWA离线缓存策略 | 弱网环境 |
| 审计日志增强 | 操作对比（before/after） | 数据追溯 |
| 多数据库支持 | PostgreSQL/MySQL | 大规模部署 |
