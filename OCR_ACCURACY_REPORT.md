## 三、失败的用例（4 个）— 全部是同一根因

| 输入姓名 | 识别结果 | 期望 | 错误原因 |
|---------|---------|------|----------|
| 赵六 | **赵六体检** | 赵六 | 正则未包含"检"字 |
| 丁一 | **丁一体检** | 丁一 | 正则未包含"检"字 |
| 司马懿 | **司马懿体** | 司马懿 | 正则未包含"体"字 |
| 张三 | **张三体检** | 张三 | 正则未包含"检"字 |

### 根因分析

当前正则表达式：
```
姓名[:：]?([^性别身份证年月日]{2,8})[性别身份证年月日]
```

停止词字符集：`性 别 身 证 年 月 日`（7 个字）

**问题**：健康证上"姓名"后面通常紧跟"体检日期"或"性别"，但正则只在遇到`性/别/身/证/年/月/日`时停止。当姓名后面直接是"体检"时：
- "检"（U+68C0）不在停止词中 → 被当成姓名的一部分
- "体"（U+4F53）不在停止词中 → 同样被吞掉

### 修复建议

在正则的停止词字符类中添加缺失的健康证常见关键字：

```
当前：[^性别身份证年月日]{2,8}
修复：[^性别身份证年月日体检发证有效至期]{2,8}
```

新增停止词：`体 检 发 机 关 有 效 至 期`（9 个字）

修复后正则：
```
姓名[:：]?([^性别身份证年月日体检发证有效至期]{2,8})[性别身份证年月日体检发证有效至期]
```

预计修复后姓名准确率：**100%**（14/14）

## 四、OCR 解析逻辑评估

### 优点
- 日期解析稳定，支持多种格式（补零/不补零/有空格）
- 有 fallback 机制（先全文本匹配，再逐行匹配）
- 模糊名字匹配（60% 字符重叠）
- 体检日期 +1 年自动计算有效期

### 需改进
1. **正则停止词不全**（上述问题）— 高优先级
2. **OCR 噪声处理**：真实 OCR 输出可能包含乱码字符，当前 `replace(/[。\u201c\u201d"]/g, '')` 只去掉了少量噪声
3. **多日期识别**：如果图片中有多个日期（体检日期、发证日期、有效期），当前只取第一个匹配的日期，可能不是体检日期
4. **姓名长度限制**：`slice(0, 4)` 硬编码最多 4 字，少数民族姓名可能超过

### 风险评估
- 当前 71% 的姓名准确率在生产环境中会导致**约 30% 的用户需要手动修正姓名**
- 日期 100% 准确，不会影响到期提醒功能
- 修复正则停止词后可将姓名准确率提升至接近 100%
